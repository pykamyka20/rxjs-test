Привет, это тестовое задание по курсу RxJs.

Задача максимально приближена к реальной, на каждом проекте мы имеем какую-либо админку, где выводятся данные и к ним добавляется фильтрация (по имени, емейлу, ид и тд)
Рассмотрим сегодня подобный случай, который поможет в будущем развернуть любое количество фильтров и максимально ускорить поиск информации

  • Создать git branch в этом проекте для себя
  • Создать lazy load модуль, который будет отображаться как стартовая страница по корневому адресу
  • Добавить в модуль компонент, реализующий следующее:
  1. Вывести в таблицу список пользователей
    3 колонки - id, email, status
    Запрос: GET https://gorest.co.in/public/v1/users
  2. Добавить 2 фильтра к таблице
    - Текстовое поле с вводом email
    - Checkbox active|inactive

    Запрос и примеры:
    https://gorest.co.in/public/v1/users?status=inactive
    https://gorest.co.in/public/v1/users?status=active
    https://gorest.co.in/public/v1/users?email=IronSpider113@gmail.com

    при пустом поле вводе разрешено отправлять пустой гет параметр, пример
    https://gorest.co.in/public/v1/users?status=inactive&email=
  3. При изменении фильтров должен автоматически отсылаться запрос на новые данные и обновлять табличку, без кнопки Submit
  4. На фильтрах должны быть обработчики, не позволяющие заспамить сервер при быстром изменении параметров
  5. Вывод на страницу осуществляется с использованием pipe async
  6. При изменении фильтров должен меняться url страницы, пример:
    Сайт DNS, при выборе фильтров изменяется url - /planshety/?stock=1&price=3999-4000&brand=apple, это позволяет сохранять выбор после обновления страницы, а так же делиться ссылкой с предвыбранным набором
  7. В обратном порядке, при нажатии f5 поле емейла и галочка active должны заполняться в соответствии с параметрами в url

Заметки и подсказки

  • * В проекте установлена библиотека, проверяющая код на общемировые стандарты написания - eslint. Сейчас у нас она используется в тестовом варианте, правила еще могут дополниться, но в будущем на каждом проекте будем писать с ней.

    Как работает - если вы написали код с ошибочным синтаксисом (пример: создали переменную через let, а можно было сделать const, поставили лишние табы и тд), среда разработки автоматически подчеркнет ошибку и по возможности предоставит вариант ее исправления, так же при сборке проекта код приведется автоматически в порядок, тем самым проект будет выглядеть всегда одинаковое, независимо от того, кто его писал

    Чтобы авматически проверить проект на корректной и исправить ошибки можно воспользоваться командой npm run lint
    Проект считается завершенным, когда выполняется без ошибок команда npm run build

    почитать подробнее о правилах можно на: https://github.com/airbnb/javascript
    https://eslint.org/
